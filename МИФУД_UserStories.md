# МИФУД — User Stories

---

## US-01: Регистрация

**Как** новый студент,
**я хочу** зарегистрироваться в системе,
**чтобы** получить доступ к оформлению заказов в столовой.

**Критерии приёмки:**
1. Студент может открыть страницу регистрации
2. Форма содержит поля: имя, email, пароль
3. При вводе уже существующего email система выводит сообщение об ошибке
4. После успешной регистрации студент перенаправляется на страницу входа

---

## US-02: Вход в систему

**Как** зарегистрированный пользователь,
**я хочу** войти в систему по email и паролю,
**чтобы** получить доступ к своему аккаунту.

**Критерии приёмки:**
1. Пользователь может открыть страницу входа
2. Форма содержит поля: email, пароль
3. При неверных данных система выводит сообщение об ошибке
4. После успешного входа пользователь перенаправляется на главную страницу
5. Сотрудник столовой получает доступ к административным страницам

---

## US-03: Выход из системы

**Как** авторизованный пользователь,
**я хочу** выйти из аккаунта,
**чтобы** другие не имели доступа к моим данным.

**Критерии приёмки:**
1. Кнопка «Выйти» доступна в навигации на всех страницах
2. После нажатия сессия завершается
3. Пользователь перенаправляется на страницу входа
4. Доступ к защищённым страницам после выхода закрыт

---

## US-04: Просмотр меню

**Как** студент,
**я хочу** просматривать меню столовой,
**чтобы** выбрать блюда для заказа.

**Критерии приёмки:**
1. Студент может открыть страницу меню
2. Отображаются только доступные блюда
3. Каждая карточка содержит название, описание, цену и фото
4. Студент может фильтровать блюда по категории
5. Студент может искать блюда по названию — результаты обновляются динамически

---

## US-05: Добавление блюда в корзину

**Как** студент,
**я хочу** добавлять блюда в корзину прямо из меню,
**чтобы** собрать заказ перед оформлением.

**Критерии приёмки:**
1. На каждой карточке блюда есть кнопка «Добавить»
2. После нажатия блюдо добавляется в корзину
3. Счётчик корзины обновляется
4. Нельзя добавить более 10 блюд в один заказ
5. При достижении лимита система уведомляет студента

---

## US-06: Оформление заказа

**Как** студент,
**я хочу** оформить заказ из корзины с указанием времени получения,
**чтобы** столовая приготовила еду к нужному моменту.

**Критерии приёмки:**
1. Студент может перейти к оформлению из корзины
2. Система отображает состав заказа и итоговую сумму
3. Поле времени получения обязательно для заполнения
4. Студент может добавить комментарий к заказу
5. Нельзя оформить более 3 активных заказов одновременно
6. После подтверждения заказ сохраняется в системе
7. Студент перенаправляется на страницу статуса

---

## US-07: Бронирование стола при заказе

**Как** студент,
**я хочу** забронировать стол при оформлении заказа,
**чтобы** иметь гарантированное место в столовой к времени получения.

**Критерии приёмки:**
1. На странице оформления заказа есть чекбокс «Забронировать стол»
2. При установке чекбокса после подтверждения заказа открывается страница выбора стола
3. Система отображает только свободные столы на выбранное время
4. После выбора стола бронь привязывается к заказу
5. Время брони соответствует времени получения заказа
6. При отсутствии свободных столов система уведомляет студента

---

## US-08: Просмотр статуса заказа

**Как** студент,
**я хочу** видеть текущий статус своих заказов,
**чтобы** знать когда приходить за едой.

**Критерии приёмки:**
1. Студент может открыть страницу статуса заказов
2. Отображаются все активные заказы студента
3. Для каждого заказа видны: номер, состав, сумма, время получения и статус
4. Если к заказу привязана бронь — отображается номер стола
5. При отсутствии заказов система предлагает перейти в меню

---

## US-09: Отмена заказа

**Как** студент,
**я хочу** отменить заказ,
**чтобы** не платить за еду которую я не заберу.

**Критерии приёмки:**
1. Кнопка «Отменить» доступна для заказов которые ещё не переданы в работу
2. Для заказов которые уже готовятся или выданы кнопка отмены недоступна
3. После отмены заказ помечается как отменённый
4. Если к заказу была привязана бронь — она также отменяется
5. Страница статуса обновляется после отмены

---

## US-10: Просмотр своих броней

**Как** студент,
**я хочу** видеть список своих броней,
**чтобы** знать какой стол за мной закреплён.

**Критерии приёмки:**
1. Студент может открыть страницу броней
2. Отображаются активные брони студента
3. Для каждой брони видны: номер стола, дата и время, статус и номер связанного заказа
4. При отсутствии броней система отображает соответствующее сообщение

---

## US-11: Добавление блюда в меню

**Как** сотрудник столовой,
**я хочу** добавлять новые блюда в меню,
**чтобы** студенты могли их заказывать.

**Критерии приёмки:**
1. Сотрудник может открыть страницу управления меню
2. Форма добавления содержит поля: название, описание, цена, категория, фото
3. Название и цена — обязательные поля
4. После сохранения блюдо появляется в меню
5. При незаполненных обязательных полях система выводит ошибку

---

## US-12: Скрытие блюда из меню

**Как** сотрудник столовой,
**я хочу** скрывать блюда из меню не удаляя их,
**чтобы** временно убирать недоступные позиции с сохранением данных.

**Критерии приёмки:**
1. У каждого блюда в панели управления есть кнопка «Скрыть»
2. После нажатия блюдо перестаёт отображаться в меню для студентов
3. Данные блюда сохраняются в системе
4. Блюдо можно вернуть в меню в любой момент

---

## US-13: Смена статуса заказа

**Как** сотрудник столовой,
**я хочу** менять статус заказов,
**чтобы** информировать студентов о готовности их заказа.

**Критерии приёмки:**
1. Сотрудник может открыть страницу управления заказами
2. Отображаются все активные заказы с текущими статусами
3. Доступна последовательная смена статуса от принятого до выданного
4. Любой активный заказ можно отменить
5. Недопустимые переходы статусов заблокированы
6. Студент видит обновлённый статус на своей странице

---

## US-14: Управление столами

**Как** сотрудник столовой,
**я хочу** добавлять столы и управлять их доступностью,
**чтобы** студенты могли бронировать только реально доступные места.

**Критерии приёмки:**
1. Сотрудник может открыть страницу управления столами
2. Отображается список всех столов с их статусами
3. Сотрудник может добавить стол с указанием номера и количества мест
4. Номер стола должен быть уникальным — при дублировании система выводит ошибку
5. Сотрудник может изменить доступность существующего стола
6. Изменения сразу отражаются на странице выбора стола для студентов

---

## US-15: Автоматический подсчёт суммы заказа

**Как** студент,
**я хочу** видеть итоговую стоимость заказа автоматически,
**чтобы** не считать сумму вручную.

**Критерии приёмки:**
1. Итоговая сумма отображается на странице оформления заказа
2. Сумма рассчитывается по каждой позиции с учётом количества
3. Итоговая сумма отображается на странице статуса заказа
